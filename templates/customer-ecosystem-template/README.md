# Customer Ecosystem Template

This template is the customer-safe starting point for a Xlooop onboarding workspace. It is designed for public discovery, authority/consent capture, source boundary control, and sterile processed knowledge.

Default state:

- customer-feedback stage;
- Watch or proposal-only operation;
- no private source ingestion;
- no GitHub remote launch;
- no team invite;
- no production/customer-success/ROI claim.

Raw customer knowledge remains in the original source until authority, consent, source scope, and do-not-ingest policy are confirmed.
